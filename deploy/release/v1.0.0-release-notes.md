## Overview

This is the first production release of **ASIMNEXUS** — a unified personal AI operating system integrating chat, identity, mesh networking, economy, and OS control into a single platform.

v1.0.0 is promoted directly from RC-2 (build 42), which passed all integration, smoke, and regression tests.

---

## Key Features

### 1. Core Chat & AI
- Universal chat interface with local-first LLM inference (Qwen3-4B GGUF model)
- Hybrid router with privacy-classified routing (public, confidential, highly_sensitive)
- Cloud trust tier enforcement: "no-cloud-for-highly-sensitive-data" security rule
- Intent detection and model tier selection via `HybridRouter`

### 2. Identity & Security
- JWT-based authentication with HS256 tokens (`asim_token`)
- 15 founder clone identities with role-based access (admin, operator)
- Dharma veto engine for safety enforcement
- Dharma Chakra power-balance constitution
- Immutable constitution with hard-lock security gates
- Biometric hardware gate integration
- Quantum vault for identity storage

### 3. Mesh Networking
- Peer-to-peer mesh with auto-discovery
- CRDT-based offline sync
- Device and node registries
- Air-gap mode support
- Bootstrap protocol for network formation

### 4. Frontend (React PWA)
- Responsive single-page application with 9 core routes
- 6 theme variants (deep-space, aurora, government, corporate, medical, minimal)
- 6 universe modes (personal, family, community, company, government, global)
- Service Worker for PWA offline caching
- WebSocket support for real-time chat
- E2E test suite with Playwright

### 5. Economy & Marketplace
- Agent contract system
- Job marketplace
- MCP protocol support

### 6. OS Control Layer
- Personal OS, World OS, and OS Control Panel interfaces
- Hardware abstraction bridge
- Docker sandbox and WASM sandbox for secure execution

### 7. Security & Governance
- Multi-layered security framework (`security/access`, `security/audit`, `security/mtls`, `security/vault`)
- Consent manager for data privacy
- OCR engine for identity verification
- Power balance constitution with immutable rules

### 8. Monitoring & Observability
- Kubernetes-style health probes (`/health/live`, `/health/ready`, `/health/status`)
- OpenTelemetry integration in `monitoring/opentelemetry/`
- Metrics dashboard and observability dashboard

## Known Issues

1. **JWT Fallback Secret**: [`backend/router.py:64-70`](../backend/router.py#L64) — When `ASIM_JWT_SECRET` env var is not set, the router falls back to a hardcoded development secret (`asimnexus-super-secret-jwt-key-2026`). This MUST be configured before production deployment.
2. **psutil Dependency**: [`backend/health.py:196-208`](../backend/health.py#L196) — System metrics (CPU, memory, disk) are silently unavailable if `psutil` is not installed. Install via `pip install psutil` for full monitoring.
3. **Model File Size**: The Qwen3-4B GGUF model requires ~4GB+ disk space and ~8GB RAM for inference. Ensure deployment host meets hardware requirements.
4. **SQLite Database**: Default database is file-based (`data/asim_core.db`). For horizontal scaling, migrate to PostgreSQL (adapter available at [`backend/postgres_adapter.py`](../backend/postgres_adapter.py)).
5. **E2E Test Flakiness**: Some Playwright tests show navigation failure artifacts in `frontend/react/test-results/` — these may be timing-sensitive and should be re-run in CI.

## Rollback Instructions

If rollback is required:

```bash
# 1. Identify previous release in releases.json
# 2. Deploy the RC-2 or RC-1 Freeze image
docker pull asimnexus/backend:1.0.0+build42-rc2
docker stop asimnexus-backend
docker run -d --name asimnexus-backend-rollback asimnexus/backend:1.0.0+build42-rc2

# 3. Update releases.json to mark previous release as current
# 4. Verify health endpoints
curl http://localhost:8000/health/status
```
